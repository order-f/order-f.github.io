The code

Even though the math is hard, and the picture complex, coding the fractal plotter is actually really easy. That must be the case, for the sole purpose of nature creating fractals is for it to be lazy. It is a use of scale-invariance to populate every scale of reality with the same stuff. That’s the underlying reason why our code implementation can be so short - the only formula to seed the algorithm is f(z)=z2+cf(z)=z^2+cf(z)=z2+c﻿ - and the fractal grows itself, engraving its own image into every layer of magnification.

We humans like looking at fractals, and symmetric objects in general, for the same reason that nature like producing them. Their symmetries means that it takes our brains much less load to perceive and store them, for we would only have to store the smallest repeating unit along with a spatial map of how to map this unit all over the perceptual region one such object occupies to recover it. Symmetrical objects can also be temporal. Apart from an omnipresent discrete time-translational symmetry in musical pieces, music like fugue also frequently exhibit fractal symmetry, in that a motif is stretched out to spans a longer time while other motifs are played simultaneously. The Alberti bass also embodies a tree-like fractal structure, though one that consists merely of two layers.

There’s no actual fractals in reality, likewise, we cannot plot actual fractals. For fractals are mathematical ideals defined by assuming that the self-similarity we see on the first few macroscopic layers does indeed turtle all the way down.

@np.vectorize # vectorize the operator so it can be applied on a meshgrid of inputs
def get_rate_of_divergence(x,y): # a function for coloring point deeper the more
                                 # iterations they need to violate the threshold
    c=x+y*1j # looping c over the meshgrid
    f=lambda z: z**2+c # define the seed function

    M=10000 # a threshold above which to treat orbit as reaching infty
    max_tries=100 # a threshold above which to treat orbit as bounded

		w=0 # starts at 0
    counter=0
    while counter<max_tries: # progressive deepening of color
        w=f(w) # orbit z
        if np.abs(w)>M: # stop at this color intensively if orbit diverges through
		                # the bound
            break
        counter+=1 # deepen color by 1
    return counter # return the intensity by which the input c is to be colored
    
nx,ny=1000,1000 # set the mesh of the sampling
x,y=np.linspace(-1.5,1.5,nx),np.linspace(-1.5,1.5,ny) # create meshgrid
xv,yv=np.meshgrid(x,y)

g=get_rate_of_divergence(xv,yv) # get a tensor of output color intensities

plt.figure(dpi=500) # set resolution
i=plt.imshow(g,cmap="PuRd",extent=[x[0],x[-1],y[0],y[-1]]) # choose a nice color
plt.colorbar() # show the map from counter to the resulting color for reference

Compactness of Mandelbrot set

Compactness is a way to constrict flow to infinity, and is equivalent as being closed and bounded by Heine-Borel theorem. Compactness is an essential property in analysis and often the first thing mathematicians look for when given a new object, owning to the abundance of theorems having compactness in the hypothesis, with one such example being that maximum principle for holomorphic functions only works solely on relatively compact subdomain. Below, I give a proof.

Bounded. The Mandelbrot set lies within the closed disk B(0;2)‾\overline{B(0;2)}B(0;2)​﻿ of radius 2 (called escape radius), which is same as saying any ccc﻿ with modulus above 2 give an unbounded orbit {fcn(0)}n\{f^n_c(0)\}_n{fcn​(0)}n​﻿ of zero.
